# 每日日报 HTML 填报工具

## 入口

GitHub Pages 静态版：

```text
https://你的用户名.github.io/wechat-daily-report/
```

静态版可以填写、预览、复制，也能在当前浏览器里自动计算；但数据保存在每个人自己的浏览器里，不能共享填写人数和全员累计。

共享版本地服务：

```bash
cd /Users/xueqing/Documents/Codex/outputs/wechat-daily-report-html
node server.js
```

打开：

```text
http://127.0.0.1:8778/
```

如果要让同一个局域网里的其他人访问，需要把 `127.0.0.1` 换成本机局域网 IP，并确保网络允许访问 8778 端口。

线上多人共享版需要把 `server.js` 部署到支持 Node 的服务器，或把数据层改成飞书多维表格、Supabase、Firebase 等统一数据库。只有这样，所有人的提交才会进入同一份数据。

## 页面流程

1. 首页选择身份：绿色、蓝色、紫色、橙色。
2. 进入填写页，只看到自己负责的字段。
3. 每个字段下面会显示上一次填写的数据，方便参考。
4. 提交后进入填写进度页，能看到已提交人数。
5. 点击进入复制预览，查看固定格式日报。
6. 点击顶部复制，直接粘贴到微信群。

## 当前口径

- 所有人只填写当日数据，不填写本月累计。
- 绿色：广播、百度、大众点评、其他。百度/大众点评/其他会合并成最终播报里的「官网及其他」当日数据。
- 蓝色：抖音/本地推、腾讯朋友圈。最终播报里的本月本地推、腾讯朋友圈数据由每天当日数据自动累加。
- 紫色：当日派单、当日签单、明日预计派单。
- 橙色：当日自然进店、当日自然签单。
- 月累计使用历史累计底数加后续每日数据自动计算，最终播报格式保持不变。
